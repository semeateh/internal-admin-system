# internal-admin-system

公司内部后台管理系统原型，当前包含流程管理模块。

## 运行方式

这是一个纯静态原型，可以直接打开 `index.html` 预览，也可以在项目根目录启动本地静态服务：

```powershell
python -m http.server 8000 --bind 127.0.0.1
```

然后访问：

```text
http://127.0.0.1:8000/index.html
```

## 后续模块

- 流程管理
- 流程模板配置
- 通知机器人配置
- 文档归档
- 用户与权限

## 默认角色

- 开发者：最高系统维护角色，可维护人员、角色、权限、系统配置，并可重置员工密码。
- 管理员：业务管理角色，可管理流程模板、发起流程、处理流程和查看流程。
- 员工：普通流程使用角色，可发起、处理和查看流程。

## 数据库设计约定

- 权限判断使用稳定的 `permission_code`，中文权限名只用于页面展示。
- 流程模板版本号 `version_code` 全局唯一，由后端统一生成，例如 `20260514001`。
- 已发布且已有实例的模板记录不可原地修改步骤、字段、负责人；模板调整必须新建版本记录。
- 流程实例只关联创建时的模板版本记录，因此旧实例不会受新模板版本影响。
- 发起流程时，后端根据模板步骤参与人配置展开员工、组、角色、部门，生成实际待办任务。

## UI 约定

后续页面和组件优先使用 Element UI 的视觉规范、主色 `#409EFF` 和组件类名，例如 `el-button`、`el-button--primary`、`el-input__inner`、`el-tag`、`el-card`。自定义样式只用于布局和业务模块差异。
